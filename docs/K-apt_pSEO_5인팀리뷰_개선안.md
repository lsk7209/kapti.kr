## 1. 윤법무 (법무) — 신규

### 1-1. 공공누리 라이선스 유형 = 빌드 전 하드 kill ⚠️⚠️
- data.go.kr 데이터가 공공누리 **제1유형(출처표시)**이면 상업적 이용·변경 허용 → OK.
- **변경금지·상업금지** 조건이 붙은 유형이면 가공+광고 수익화 자체가 불가.
- v0.1은 "출처표시"만 명시, **유형 확인을 게이트로 미설정** → 치명적 누락.
- **권고**: Gate 0.5 0순위에 **라이선스 유형 명시 확인** 추가. 제한형이면 즉시 kill.

### 1-2. 단정적 판정의 법적 리스크
- 특정 단지를 "관리비 폭리 / 충당금 부족"으로 단정 → 관리주체·입주자대표회의 대상 명예훼손·업무방해 소지(한국법은 사실 적시도 성립 가능).
- 면책 고지로 방어 불충분.
- **권고**: 평가·단정 전면 배제, **"공공데이터 수치 + 중립 맥락"만**.

### 1-3. 표시광고법 (순위·비교 표기)
- "TOP/최저/최고" 표기 시 기준·산정방식·출처 병기 의무.

### 1-4. 정보 vs 자문 경계
- "사라/사지 마라" 식 권유 금지(공인중개사법 등). 정보 제공까지만.

### 1-5. 개인정보
- 기본정보 API에 관리사무소 연락처 등 개인식별정보 혼입 시 제거. 필드 점검을 Gate 0.5에 포함.

## 2. 조엔지 (제품·엔지니어링) — 신규

### 2-1. 매월 또래 재계산 파이프라인 ⚠️
- 신규 월 관리비마다 또래 분포(시군구×난방×세대수×부대시설) 재집계 + 2만 단지 백분위 갱신 필요.
- Edge SQLite(Turso)에서 실시간 group-by 백분위는 부적합.
- **권고**: GitHub Actions 배치 사전계산 → **결과 테이블에 머티리얼라이즈**, ISR/SSR은 그 결과만 읽음.

### 2-2. 통제변수 추가 → 비교군 붕괴
- 부대시설·난방 통제 시 시군구 또래가 한 자릿수로 떨어지는 단지 속출 → 백분위 노이즈.
- **권고**: **최소표본 n 임계값(예 n≥30) 미달 시 상위 지리(시도→전국 동일 난방)로 계층 폴백** 스펙화. 폴백 시 비교 단위에 맞춰 진단 카피 동적 변경.

### 2-3. 데이터 볼륨
- 단지 2만 × 월별 시계열(수년) × 항목 수십 개 = 수백만~수천만 행.
- **권고**: Turso 용량·쿼리 한계를 Gate 0.5에서 확인. 관리비 원시 vs 집계 테이블 분리.

### 2-4. slug 충돌
- 동명 아파트 다수 → **kaptCode를 slug에 포함**해 유니크 보장.

### 2-5. ISR 재검증
- 관리비 월 갱신 시 영향 페이지 on-demand revalidation 트리거 설계.
